In emergency situations such as accidents or sudden falls, response time becomes the critical factor between life and death.
However, most existing safety systems rely heavily on manual intervention or internet connectivity, both of which can fail during high-risk scenarios.

So, we asked ourselves a fundamental engineering question:
Can we design a system that automatically detects danger, operates without internet dependency, and still gives control to the user?

This led us to our solution —
“Life-Pulse Guardian”

Our objective is to develop a smart, reliable, and cost-effective safety system that:

Automatically detects sudden falls
Sends SOS alerts with real-time location
Operates independently of network connectivity
Minimizes false alerts using intelligent pre-alert logic
System Components

Our system integrates the following hardware:

MPU6050 (Accelerometer + Gyroscope)
→ Detects sudden motion and impact (threshold set at ~4g force)
NEO-6M GPS Module
→ Provides real-time latitude and longitude
→ Works via satellite communication — no internet required
Arduino Uno
→ Acts as the central processing unit controlling all operations
Push Buttons (2)
→ Manual SOS trigger
→ Alert cancellation
Buzzer + LED
→ Provide immediate alert indication
LCD Display
→ Displays system status and alerts in real time


Working Principle (Step-by-Step)

Step 1: Continuous Monitoring
The MPU6050 continuously measures acceleration along the X, Y, and Z axes and calculates the resultant acceleration.

Step 2: Fall Detection
If the acceleration exceeds a predefined threshold (~4g), the system classifies it as a potential fall event.

Step 3: Pre-Alert Stage (Key Innovation)
Instead of triggering an immediate SOS:

A 5-second delay window is introduced
The user can cancel the alert using a button

This significantly reduces false alarms, such as accidental drops.

Step 4: SOS Activation
If no cancellation occurs:

The GPS module retrieves real-time coordinates
An SOS signal is triggered with location data

Step 5: Manual Override
At any moment, the user can manually activate SOS, independent of fall detection.

Now let’s understand the working of our system step by step, which is the most critical part.
Step 1: Continuous Monitoring

The MPU6050 continuously measures acceleration in all three axes.

The Arduino calculates the resultant acceleration, which ensures that detection is independent of device orientation.

This is important because in real-world scenarios, the device can be in any position.

Step 2: Fall Detection

When a human body falls, there is a sudden spike in acceleration.

If the calculated acceleration exceeds ~4g, the system identifies it as a potential fall event.

Step 3: Pre-Alert Stage (Key Innovation)

This is one of the most important features of our project.

Instead of immediately triggering an SOS:

A 5-second delay window is introduced
The buzzer and LED are activated
The LCD displays a warning message

During this time, the user can cancel the alert.

👉 This step is crucial because it reduces false alerts caused by:

Accidental drops
Sudden movements
External vibrations
Step 4: Decision Logic

After the delay:

If the user cancels → system resets
If no response → system assumes emergency

This ensures that alerts are triggered only when truly needed.

Step 5: SOS Activation

Once confirmed:

GPS module retrieves real-time coordinates
SOS alert is triggered
Buzzer and LED continue to indicate emergency state
Step 6: Manual Override

Even if no fall is detected:

The user can manually press a button
SOS will be triggered instantly

This ensures complete user control.

7. GPS-Based Location Tracking

Once an emergency is confirmed, the system activates the NEO-6M GPS module to obtain real-time location data. The GPS module communicates with satellites and calculates the device’s position using triangulation.

Since GPS works independently of internet connectivity, the system can function effectively even in remote or network-dead zones, making it highly reliable in critical scenarios.

8. SOS Activation

After retrieving the location coordinates, the system triggers the SOS alert. The coordinates are displayed on the LCD and can be transmitted to external systems (such as GSM or LoRa modules in future enhancements).

Simultaneously, the buzzer and LED continue to indicate that the system is in emergency mode.

9. Manual Override Feature

In addition to automatic detection, the system includes a manual SOS button. This allows the user to trigger an emergency alert at any time, regardless of motion detection.

This dual-trigger approach ensures that the system remains functional even in situations where a fall is not detected but assistance is still required.

Future Scope: 
In our future scope, we plan to enhance the system by integrating a fire detection mechanism using an NTC 100K thermistor. This sensor works on the principle that 
its resistance decreases as temperature increases, allowing us to accurately monitor environmental temperature. Although the thermistor can withstand temperatures up 
to 300°C, we will set a practical threshold of around 100°C, since temperatures beyond this level typically indicate fire hazards or unsafe conditions. 
The sensor will be connected through a voltage divider circuit, and the Arduino will continuously monitor temperature changes. If the temperature exceeds the set threshold,
the system will automatically trigger an SOS alert, similar to fall detection. This addition transforms our project from a single-purpose safety device into a multi-hazard 
detection system, making it more effective in industrial, rescue, and disaster management applications.

In addition to fire detection, we also plan to integrate a water level sensor to detect flood-like situations. This sensor works by sensing the presence or level of
water through changes in conductivity or resistance across its probes. Under normal dry conditions, the sensor outputs minimal signal, but when water comes in contact, 
the conductivity increases, generating a measurable output. The Arduino will continuously monitor this signal, and if water is detected beyond a predefined level, the
system will identify it as a potential flood or water hazard. Similar to other emergency conditions, an SOS alert will be triggered automatically. This enhancement will
allow our system to respond to environmental disasters such as floods or water leakage, further expanding its capability into a comprehensive multi-hazard safety system
suitable for real-world applications.
